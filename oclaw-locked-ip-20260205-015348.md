# Azure VM Network Lock Documentation

**Created:** 2026-02-05 01:53:48
**VM Name:** `oclaw2026linux`
**Resource Group:** `RG_OCLAW2026`
**Public IP:** `20.81.190.88` (Static)

---

## Problem Summary

The VM's public IP address was being **automatically detached** by Azure service principal `7355f99c-0211-455d-aa02-4a559687ae60`, causing SSH connectivity failures.

### Issues Identified

1. **Public IP Detachment:** The public IP `oclaw2026linux-ip` (20.81.190.88) was disconnected from the VM's network interface
2. **Missing Subnet NSG Rule:** Subnet-level Network Security Group was missing SSH allow rule
3. **Automated Service Principal:** Service principal was automatically:
   - Deallocating (stopping) the VM at ~05:52 UTC daily
   - Detaching network resources
   - Clearing/resetting NSG configurations

### Network Architecture

```
VM: oclaw2026linux
  ├─ NIC: oclaw2026linux787_z3
  │   ├─ Private IP: 172.20.0.4
  │   ├─ Public IP: 20.81.190.88 (oclaw2026linux-ip) ← Was being detached
  │   └─ NIC NSG: oclaw2026linux-nsg ✓ Has SSH rule
  │
  └─ Subnet: snet-eastus2-1 (in vnet-eastus2)
      └─ Subnet NSG: vnet-eastus2-snet-eastus2-1-nsg-eastus2 ← Added SSH rule
```

---

## Solution Implemented

### 1. Reattached Public IP

```bash
az network nic ip-config update \
  --nic-name oclaw2026linux787_z3 \
  --resource-group RG_OCLAW2026 \
  --name ipconfig1 \
  --public-ip-address oclaw2026linux-ip
```

### 2. Added SSH Rule to Subnet NSG

```bash
az network nsg rule create \
  --nsg-name vnet-eastus2-snet-eastus2-1-nsg-eastus2 \
  --resource-group RG_OCLAW2026 \
  --name AllowSSH \
  --priority 100 \
  --destination-port-ranges 22 \
  --protocol Tcp \
  --access Allow \
  --direction Inbound
```

### 3. Created Resource Locks (Permanent Fix)

Two `CanNotDelete` locks were created to prevent automated detachment:

#### Lock 1: Public IP Address Lock
```bash
az lock create \
  --name "DoNotDetach-PublicIP" \
  --resource-group RG_OCLAW2026 \
  --resource oclaw2026linux-ip \
  --resource-type Microsoft.Network/publicIPAddresses \
  --lock-type CanNotDelete \
  --notes "Prevent auto-detachment by service principal"
```

**Lock ID:**
```
/subscriptions/c34d54e5-7eb9-4ba8-8424-c6ab8635ccdc/resourcegroups/RG_OCLAW2026/providers/Microsoft.Network/publicIPAddresses/oclaw2026linux-ip/providers/Microsoft.Authorization/locks/DoNotDetach-PublicIP
```

#### Lock 2: Network Interface Lock
```bash
az lock create \
  --name "DoNotModify-NIC" \
  --resource-group RG_OCLAW2026 \
  --resource oclaw2026linux787_z3 \
  --resource-type Microsoft.Network/networkInterfaces \
  --lock-type CanNotDelete \
  --notes "Prevent NIC configuration changes"
```

**Lock ID:**
```
/subscriptions/c34d54e5-7eb9-4ba8-8424-c6ab8635ccdc/resourcegroups/RG_OCLAW2026/providers/Microsoft.Network/networkInterfaces/oclaw2026linux787_z3/providers/Microsoft.Authorization/locks/DoNotModify-NIC
```

## What the Locks Prevent

| Action | Before Locks | After Locks |
|--------|--------------|-------------|
| View resources | ✓ Allowed | ✓ Allowed |
| Start/Stop VM | ✓ Allowed | ✓ Allowed |
| SSH to VM | ✓ Allowed (when running) | ✓ Allowed |
| Delete public IP | ✓ Allowed | ✗ **BLOCKED** |
| Detach public IP from NIC | ✓ Allowed | ✗ **BLOCKED** |
| Modify NIC IP config | ✓ Allowed | ✗ **BLOCKED** |
| Delete NIC | ✓ Allowed | ✗ **BLOCKED** |

## Troubleshooting

### If SSH Still Fails

1. **Check VM is running:**
   ```bash
   az vm show -n oclaw2026linux -g RG_OCLAW2026 -d --query powerState -o tsv
   ```

2. **Check public IP is attached:**
   ```bash
   az network public-ip show -n oclaw2026linux-ip -g RG_OCLAW2026 \
     --query "ipConfiguration.id" -o tsv
   ```

3. **Check both NSG rules exist:**
   ```bash
   # NIC NSG
   az network nsg rule show --nsg-name oclaw2026linux-nsg -g RG_OCLAW2026 --name AllowSSH

   # Subnet NSG
   az network nsg rule show --nsg-name vnet-eastus2-snet-eastus2-1-nsg-eastus2 \
     -g RG_OCLAW2026 --name AllowSSH
   ```

4. **Check effective NSG rules:**
   ```bash
   az network nic list-effective-nsg --ids /subscriptions/c34d54e5-7eb9-4ba8-8424-c6ab8635ccdc/resourceGroups/rg_oclaw2026/providers/Microsoft.Network/networkInterfaces/oclaw2026linux787_z3
   ```

### If Locks Prevent Legitimate Changes

To temporarily remove a lock:
```bash
# Remove public IP lock
az lock delete --name "DoNotDetach-PublicIP" -g RG_OCLAW2026 \
  --resource oclaw2026linux-ip \

# Make your changes

# Recreate lock
az lock create --name "DoNotDetach-PublicIP" -g RG_OCLAW2026 \
  --resource oclaw2026linux-ip \
  --resource-type Microsoft.Network/publicIPAddresses \
  --lock-type CanNotDelete
```
