Subject: memory handling

Hi,

Here’s how I prevent “context rot” in my memory system:

- Separate “journal” from “truth”
  - Daily notes are chronological logs (allowed to be messy/temporary).
  - Long-term memory is curated: only stable preferences, enduring decisions, evergreen facts.

- Write memories as policies, not anecdotes
  - Prefer: “When creating/updating ClickUp tasks, use real line breaks + bullets (avoid literal \\n).”
  - Avoid: one-off incident summaries.

- Add scope/expiry when it’s likely to change
  - “Temporary until X”, “applies only to project Y”, “valid as of YYYY-MM”.

- Prefer pointers over copies
  - Store links/paths to the canonical artifact instead of duplicating long content.

- Prune regularly
  - Promote only durable items into long-term memory; leave the rest in daily logs.

- Resolve conflicts explicitly
  - If a new rule contradicts an old one, update/replace the old rule rather than accumulating both.
